    このツールについて

    ■ 概要
    このツールは、ranked譜面を作成する mapper や、modding を行う Beatmap Nominator 向けの osu!taiko modding 支援ツールです。
    また、一部には BN evaluation を行う NAT 向けの機能も実装しています。
    ■ 背景
    Ranked譜面を作成する際には、一般的に AiMod、 Mapset Verifier、 Mapset Verifier MVTaikoChecks などのチェックツールを使用します。しかし、これらのツールで検出できる内容は完全ではありません。

    例えば Taiko Checklist V1.2 by iRedi を見ると、これらのツールでは検出できないチェック項目も存在することが分かります。

    本ツールは、主に Mapset Verifier ではチェックできない項目を自動で確認するためのツールです。
    基本的には、Mapset Verifier と本ツールの両方を併用することを想定しているため、Mapset Verifier で検出可能な項目は本ツールでは基本的に検出しません。

    また、本ツールには一部の AiMod に近い機能も実装しています。
    （⚠ 全てではありませんので最終的な AiMod によるチェックは必須です）
    「全diffを一括で確認できる AiMod」のような使い方もできます。
    ■ 参考
    • Ranking Criteria
    • osu!taiko Ranking Criteria
    ■ 便利な設定
    • 使用しないチェック項目は、画面右上の歯車ボタンから非表示にできます。
    • チェックボックスやタブの表示設定はブラウザに保存されます。
      そのため、次回アクセス時も前回と同じ設定で使用できます。
    • 各タブのチェック結果に応じてタブの色が変化します：警告 / エラー 判定一覧
    • ハイブリッド mapset の場合は、taiko モードの diff のみを表示します。
    ■ 注意事項
    • 本ツールと Mapset Verifier を併用しても、すべての Ranking Criteria 違反を検出できるわけではありません。
    • このツールの snap 判定は beatLength を基準にしており、meter（拍子）は考慮していません。
      そのため、1/1 snap は小節頭ではなく「1拍ごとの位置」を意味します。
    • ブラウザでページを開いたままにしている場合、ツールの更新が自動では反映されないことがあります。
      定期的にページをリロードするか、更新が反映されない場合は Ctrl + F5 で強制リロードしてください。

    注意事項 / プライバシー

    ■ チェック内容について
    本ツールの判定は、事前に定義したルールベースのチェックによって行われています。
    生成AIなどによる自動推測・自動判定は使用していません。

    例：
    • Tag のスペルチェック：内部辞書との比較によって行われます
    • 東方 Source チェック：事前に定義した作品名一覧との一致判定によって行われます
    • snap 判定：BPM line と timing 計算によって行われます
    未知の単語・特殊表記・新ジャンルなどについては、正しく判定できない場合があります。
    ■ アップロードファイルについて
    アップロードされた .osu / .osz ファイルは、ブラウザ内でのみ処理されます。

    ファイル内容が、私を含む第三者へ送信・公開されることはありません。
    また、サーバーへのアップロード・保存なども行っていません。

    チェック処理は、すべてクライアント側（JavaScript）で実行されます。

    また、本ツールのソースコードは GitHub 上で公開されています： modding-helper
    ■ アクセス解析について
    本ツールでは、利用状況の把握や改善のために Google Analytics を使用しています。

    アクセス数・閲覧ページ・利用環境などの匿名統計情報が収集される場合があります。
    ただし、アップロードされた .osu / .osz ファイルの内容が送信されることはありません。

    収集される情報は、Google Analytics の仕様に基づいて処理されます。
    ■ 免責事項
    本ツールは、osu!taiko の modding / checking を補助するためのツールです。
    表示される Warning / Error は補助情報であり、rankability や correctness を保証するものではありません。

    本ツールの使用によって発生した問題・不利益・損害などについて、開発者は責任を負いません。
    ツールの利用・判定結果の採用については、ご自身の判断と責任で行ってください。

    また、本ツールは可能な限り正確な判定を目指していますが、誤検出・未検出・仕様変更への未対応などが発生する可能性があります。
    バグの報告・表記方法の改善案の提案・新機能の提案などありましたら開発者までご連絡お願い致します。

    Clap / Whistle

    ■ 背景
    Kat には 2種類（Clap / Whistle）があります。
    一部のBNは Whistle の音を好まないため、Clap に統一するのが望ましい場合があります。
    ■ 検出内容
    Circle（type=1/5）の hitsound を解析し、Clap / Whistle / 両方 の個数を表示します。

    「Whistle のタイムスタンプを表示」or 「Clap のタイムスタンプを表示」チェックボックスを有効にすると、それぞれのタイムポイントも表示されます。
    ■ 注意点
    Whistle 、Whistle + Clap の使用は unrankable ではありません。

    1msズレ

    ■ 背景
    60000 / BPM が割り切れない場合、osu!editorでのコピーペーストにより浮動小数誤差が発生し、1 ms のズレが生じることがあります。
    1 msズレは SV線の未適用や SR計算の誤差につながるため、なるべく避けたほうが良いです。

    Mapset Verifier や AiMod では ±1 ms のズレは表示されません（unrankable ではないため）。
    ■ 検出内容
    ノーツ・スライダー・スピナー が snap 位置から ±1 ms ずれている場合は警告、 ±2,3 ms ずれている場合はエラーとして検出します。

    1 ms単位での正確な snap 推定は、beatLength を用いて計算し、最終的に小数点以下を切り捨てることで行っています。 これにより、osu!editor とほぼ同じ位置の snap を推定できます。

    終点については、スピナー終点に加えてスライダー終点のズレ検出にも対応しています。
    スライダー終点は、pixelLength・SliderMultiplier・SV・BPM をもとに近似計算し、四捨五入しています。
    一部の譜面では実際の editor 上の終点時刻と完全一致しない可能性があります。

    チェック対象の snap は、1/1, 1/2, 1/3, 1/4, 1/6, 1/8, 1/12, 1/16 です。
    それ以外の snap は検出対象外です。

    なお、1/5, 1/7, 1/9 snap は、チェックボックスを有効にした場合のみ検出されます。
    ■ 注意点
    ±1 ms は unrankable ではないため、必ずしも全て修正する必要はありません。

    Kiai比較

    ■ 背景
    diffごとに kiai 総時間が異なる場合、どの区間で差があるのか分かりにくいことが多いです。
    Mapset Verifier では総時間の比較はできますが、具体的にどの区間が異なるかは確認できません。
    ■ 検出内容
    kiai 状態が一致していない区間を表示します。

    また、kiai time の終了地点が明示されていない場合も警告を行います（AiModと同一の機能）。
    ■ 注意点
    diffごとに kiai を変えること自体は unrankable ではありません。特に Guest difficulty ではよく見られます。
    ただし、同一作者の場合は特別な理由がない限り揃えることが推奨されることが多いです。

    Kiaiスナップ

    ■ 背景
    AiMod は unsnapped な kiai timing point を検出できますが、1/4 や 1/8 snap など、snap単位で kiai がズレている場合は検出できません。
    ■ 検出内容
    kiai の ON/OFF が 1/1 snap 以外 に乗っている場合を検出します。

    また、unsnapped な kiai line（1/1 snap + 1 ms など）も検出します（AiModと同一の機能）。

    二重SV

    ■ 背景
    二重 SV はどちらかが意図しない or 消し忘れであるケースが多いです。
    AiMod は同時刻の重複 SV は検出できますが、1 ms ズレの二重 SV は検出できません。
    ■ 検出内容
    指定した間隔（1 ~ 5 ms）以内にある、近すぎる SV line を検出します。

    SV ボリューム

    ■ 背景
    HitObject と同時刻に volume変更を置いた場合、プレイヤーが 1 ms 早く叩くと新しい volume が適用されない場合があります。
    そのため、volume変化を伴うSV線は、HitObject に対して 5 ms 以上前倒しして設置することが推奨されています（一般的には 5 ms, 1/16 snap, 1/8 snap 程度ズラす事が多いです）。
    ■ 検出内容
    ボリューム変更を伴う SV line が、HitObject と近すぎる場合に警告します（この検出は AiMod や Mapset Verifier では行えません）。

    検出閾値は、5 ms（丁度は含まない） 1/16 snap（丁度は含まない）から選択できます。

    15% 以上の差のみ表示 を有効にすると、volume 差が 15% 以上の際のみ表示します。

    ボリューム比較

    ■ 検出内容
    diff間の SV ボリュームを比較し、不一致がある区間を検出します。
    （この検出は AiMod や Mapset Verifier では行えません）

    5% 以上の差のみ表示 を有効にすると、volume 差が 5% 以上の区間のみを表示します。

    50 ms 以上続く区間のみ表示 を有効にすると、50 ms 以上以下の極端に短い区間を非表示にできます。

    赤線 & 緑線の一致

    ■ 背景
    同時刻の赤線（BPM line）と緑線（SV line）は、基本的に同じ設定である必要があります。
    ■ 検出内容
    ボリューム / kiai のオンオフ / サンプルセット の不一致を検出します。

    （この検出は AiMod や Mapset Verifier では行えません）
    ■ 注意点
    最初の赤線は、osu!editor の仕様上 kiai ON にすることができません。

    osu! Ranking Criteria では、譜面開始地点から kiai を有効にしたい場合、 同時刻に緑線を配置し、その緑線のみを kiai ON にすることが推奨されています。

    そのため、このチェックでは「最初の赤線」と「最初の緑線」の kiai 状態が一致しない場合については、 警告を出さないようになっています。

    また、最初の赤線と緑線では、意図的に volume を変更している場合があります。 そのタイムポイントより前は赤線の volume が適用され、それ以降は緑線の volume が適用されるためです。

    サンプルセット

    ■ 背景
    BPM線（赤線）やSV線（緑線）、および各 HitObject には、sampleset と呼ばれる効果音の種類を割り当てることができます。
    sampleset はヒット音の「音色」を決める設定で、主に以下の種類があります：
    • 1: Normal
    • 2: Soft
    • 3: Drum
    osu!taiko モードでは、大抵の場合 1: Normal のみを使用するため、それ以外の sampleset が含まれている場合は、意図しない効果音が混入している可能性があります。
    ■ 検出内容
    Normal 以外の sampleset やカスタムヒットサウンドを検出します。
    （この検出は AiMod や Mapset Verifier では行えません）

    赤線・緑線では、以下を検出します：
    • sampleset ≠ 1 (normal) → 2: Soft / 3: Drum
    • sampleIndex ≠ 0 → カスタムヒットサウンド
    ノーツ、スライダー、スピナーでは、以下を検出します：
    • normalSet > 1 → 2: Soft / 3: Drum
    • additionSet > 1 → 2: Soft / 3: Drum
    • sampleIndex ≠ 0 → カスタムヒットサウンド
    • filename が指定されている → カスタム音源
    スライダーの各エッジに設定されたヒットサウンドも検出します：
    • edgeSets.normalSet > 1 → 2: Soft / 3: Drum
    • edgeSets.additionSet > 1 → 2: Soft / 3: Drum

    スライダー感度・目盛り

    ■ 背景
    taiko モードでは、スライダー感度（SliderMultiplier）は基本的に 1.40 に設定します。

    スライダー目盛り（SliderTickRate）は通常 1 を用いますが、
    swingリズムの曲では 3 を用います。
    ■ 検出内容
    SliderMultiplier が 1.40 ではない場合に警告します

    全ての HitObject の位置から 1/3 snap の割合を計算し、割合が 30% 以上の場合は swingリズムとして扱います（1/3 snap から 1ms ズレしたノーツも考慮しています）。

    swingリズムと判定された場合は SliderTickRate = 3、それ以外の場合は SliderTickRate = 1 を推奨値として表示し、実際の値と異なる場合に警告します

    （この検出は AiMod や Mapset Verifier では行えません）。
    ■ 注意点
    SliderMultiplier = 1.40 は、osu!taiko ranked map で最も一般的に使用されている値です。

    ただし、1.40 以外の値の使用が禁止されている訳ではありません。
    例えば：
    • 高 BPM 譜面の low diff において、1.40 より低い SliderMultiplier を使用するケース （参照：Taiko Checklist V1.2 by iRedi）
    • 低 BPM 譜面で、審美性や既存 SV 構造を維持するために、 SV 値を変更せず SliderMultiplier側を調整するケース
    • Swing 楽曲において、SliderMultiplierを上げることで、 1/3・1/6 snap を視覚的に 1/2・1/4 snap に近づけ、 readability や審美性を改善するケース
    「1/3 snap の割合が 30% 以上」という基準は、このツール上で便宜的に設定した値です。
    そのため、この値を下回っている swing rhythm map や、 この値を上回っている straight rhythm map が存在する可能性があります。

    最終的には、曲のリズムや譜面内容を確認した上で判断してください。

    プチフリ

    ■ 背景
    osu! では、音源の読み込み時に一瞬のフリーズ（プチフリーズ）が発生する場合があります。

    音源開始直後に最初のノーツが配置されていると、ノーツが画面に表示されている最中、または判定枠に近い位置でこのフリーズが発生し、正しいリズムで叩くことが難しくなる可能性があります。

    そのため、最初のノーツが音源開始直後に置かれている場合は、音源ファイルの先頭に 500 ~ 1000 ms 程度の無音を挿入することが一般的に推奨されています。
    ■ 検出内容
    最初のノーツが、音源開始時点で画面内のどの位置に表示されているかを推定します。
    Circle オブジェクトのみを対象とし、Slider や Spinner は検出対象外です。

    BPM と SV から osu!taiko のスクロール速度を推定し、16:9 環境を基準として、ノーツが画面に入ってから判定枠に到達するまでの時間を計算します。

    参考： Note Speed Mechanics

    結果には、以下の情報を表示します：
    • Visible：ノーツが画面に入ってから判定枠に到達するまでの推定時間
    • Remain：音源開始時点で、判定枠までどの程度残っているかの推定割合
      （音源開始時点でのノーツ位置）
    ■ 判定基準
    • 警告：Remain ≤ 50%
    • エラー：Remain ≤ 25%
    ■ 注意点
    この判定は、16:9 環境を基準にした推定です。
    実際の見え方は、画面比率、スキン、HR の有無、osu!stable / osu!lazer の違い、PC環境などによって変わる可能性があります。

    また、プチフリの発生タイミングや強さは環境依存です。
    このチェックは「必ず問題が起こる」ことを示すものではなく、音源開始直後のノーツ配置が危険かどうかを確認するための目安として使用してください。

    メタデータ：アーティスト

    ■ 背景
    Artist / ArtistUnicode は、mapset内の全diffで基本的に一致している必要があります。

    また、feat.、vs.、CV: などの marker は、 Ranking Criteria に沿った標準表記にする必要があります。
    ■ 検出内容
    以下を検出します。
    • 全diffで Artist / ArtistUnicode が一致しているか
    • 特殊記号が推奨Romanised表記に変換されているか
    • feat.、vs.、CV:、VO: などの marker 表記が正しいか
    • カンマの後ろにスペースがあるか
    • 連続する半角スペースや全角スペースが含まれていないか
    詳細なチェック対象は、 Metadata チェックリスト を参照してください。

    メタデータ：タイトル

    ■ 背景
    Title / TitleUnicode は、mapset内の全diffで基本的に一致している必要があります。

    また、(TV Size)、(Short Ver.)、(Game Ver.) などの version marker は、 Ranking Criteria に沿った標準表記にする必要があります。
    ■ 検出内容
    以下を検出します。
    • 全diffで Title / TitleUnicode が一致しているか
    • 特殊記号が推奨Romanised表記に変換されているか
    • (Short Ver.)、(Game Ver.)、(Movie Ver.) などの title marker 表記が正しいか
    • タイトル内の (feat. Artist) 表記が正しいか
    • 連続する半角スペースや全角スペースが含まれていないか
    詳細なチェック対象は、 Metadata チェックリスト を参照してください。
    ■ 注意点
    (Extended Edit) or (Extended Ver.) の判断は、公式のExtendedであるか非公式のExtendedであるかに依存します。

    このツールでは、主に「既に似た marker があるが標準表記ではない」ケースを検出するだけです。

    メタデータ：ソース

    ■ 背景
    東方作品の Source は、一般的に 日本語タイトル + 全角スペース + 全角チルダ + 半角スペース + 英語タイトル + ピリオド
    という独特な形式で表記されることが多いです。

    例： 東方風神録　～ Mountain of Faith.
    ■ 検出内容
    まず、全diffで Source が一致しているかを確認します（Mapset Verifier と同一の機能）。

    その後、Source を東方作品名のリストと照合します。
    チェックリスト

    完全一致する場合は OK として表示します。

    東方作品と思われる Source で、既知の作品名と表記が微妙に異なる場合は、スペルミスやスペースの全角 / 半角ミスの可能性があるため警告します。
    その場合、推奨表記と参照リンクを表示します。

    また、東方 や Touhou などの汎用的な表記のみの場合も、具体的な作品名が不足している可能性があるため確認対象として表示します。
    ■ 注意点
    このチェックは、東方作品の Source 表記を補助的に確認するための機能です。
    東方作品以外の Source はチェック対象外として扱います。

    また、未知の東方作品名や、ゲーム作品以外の東方関連作品、特殊な表記については正しく判定できない場合があります。その場合は、表示された結果だけで判断せず、公式サイトや beatmap submission の内容を確認してください。

    メタデータ：タグ

    ■ 背景
    Tag と Tag の間は、半角スペース1つ で区切る必要があります。

    2つ以上の半角スペース や全角スペース が含まれている場合、AiMod や Mapset Verifier では、検知できません。
    ■ 検出内容
    まず、全diffで Tags が一致しているかを確認します（Mapset Verifier と同一の機能）。

    その後、各diffの Tags に対して、以下のスペース問題を検出します：
    • Tag と Tag の間に、2つ以上の半角スペースが含まれている場合
    • Tag と Tag の間に、全角スペースが含まれている場合
    • スペルミスの可能性がある tag
    • 関連tag / abbreviation の追加候補
    スペルチェックでは、主にジャンル名や mapping 関連の tag を対象に、1文字違い・1文字抜け・1文字多い可能性がある tag を検出します。
    例：hardcroe → hardcore, vocaliod → vocaloid

    関連tag / abbreviation 候補では、特定の tag が含まれている場合に、併記されることが多い tag を追加候補として表示します。
    例：jpop → j-pop, pop / tiebreaker → tb

    Tagのチェックリスト

    この検出は AiMod や Mapset Verifier では行えません。
    ■ 注意点
    スペルチェックや関連tag候補は、必ず修正・追加が必要なエラーではありません。
    特にジャンル名や abbreviation は譜面内容や楽曲情報によって適切な tag が変わるため、最終的には mapper / modder / BN が内容を確認して判断してください。

    また、対応している tag の種類は限定的であり、全てのスペルミスや関連tagを検出できるわけではありません。

    その他：プレビューポイント

    ■ 背景
    mapset 全体のオフセットを変更した場合、HitObject や TimingPoint は修正していても、PreviewTime の変更を忘れることがあります。
    ■ 検出内容
    PreviewTime が snap からずれている場合、warning として表示します
    （この検出は AiMod や Mapset Verifier では行えません）。
    ■ 注意点
    PreviewTime が snap 上にないことは、必ずしも unrankable を意味するわけではありません。

    ただし、オフセット変更後の修正漏れを見つける手がかりになるため、warning が出た場合は意図した位置かどうか確認してください。

    その他：BGオフセット

    ■ 背景
    [Events] セクション内の background event では、背景画像のファイル名と表示位置のオフセットを指定できます。

    例： 
    [Events]
    
//Background and Video events
    
0,0,"BG.jpg",0,40
    background event は、以下の形式で記述されます：
    0,0,filename,xOffset,yOffset

    xOffset / yOffset が 0,0 の場合、記述は省略されることがあります。
    ■ 検出内容
    mapset 内の各 difficulty について background event を読み取り、同じ背景画像ファイルを使用している difficulty 間で xOffset / yOffset が一致しているかを確認します。

    同じ background file を使用しているにもかかわらず、diff 間で offset が異なる場合は warning として表示します。

    この検出は AiMod や Mapset Verifier では行えません。
    ■ 注意点
    異なる background file を使用している difficulty 同士は比較対象外です。

    例えば、Kantan では BG.jpg、Muzukashii では BG2.jpg を使用している場合、 それぞれ別の背景画像として扱うため、offset が異なっていても warning にはしません。

    また、同じ background file で offset が異なる場合でも、意図的な演出である可能性があります。
    warning が出た場合は、修正漏れか意図的な差分かを確認してください。

    その他：点滅・てんかん警告

    ■ 背景
    osu! では、繰り返しのストロボ発光や高速な点滅演出が含まれる storyboard / video / kiai effect を使用する場合、 epilepsy warning の表示が必要になる場合があります。

    特に osu!taiko の kiai effect では：
    • hit object flash
    • star fountain
    • 画面側面の flash
    などの演出が BPM に応じて高速化するため、 高BPM や短時間の kiai flash を多用した譜面では、視覚的刺激が強くなる可能性があります。

    本機能は、そのような「高頻度の点滅演出」の可能性を補助的に確認するための機能です。
    ■ 検出内容１：Kiai Flash 頻度
    Kiai の OFF → ON 切り替え頻度を解析し、 短時間に繰り返される Kiai flash を検出します。

    このチェックでは：
    • 2 Hz 以上：Caution
    • 3 Hz 以上：Warning
    として表示します。

    例えば： 
    ON 100ms
OFF 233ms
ON 100ms
OFF 233ms
ON 100ms
    のように、短時間で繰り返し Kiai が ON になる場合は、 高頻度 flash として確認対象になります。

    Kiai flash は、Kiai の OFF → ON 切り替え間隔から推定しています。
    一般的に、3 Hz 以上の点滅は photosensitive epilepsy の観点から注意対象とされているため、 本ツールでは 1秒間に3回以上の Kiai ON が発生する場合を Warning として扱います。
    ■ 検出内容２：高BPM Kiai
    Kiai 中の BPM を解析し、 高BPM による高速な kiai flash の可能性を確認します。

    osu!taiko の kiai effect は BPM に応じて flash 頻度が増加するため、 高BPM の Kiai は視覚的刺激が強くなる場合があります。

    本ツールでは：
    • 240 BPM (4 Hz) 以上：Caution
    • 300 BPM (5 Hz) 以上：Warning
    として表示します。

    表示では、以下の情報を出力します：
    • BPM：その Kiai 区間の BPM
    • Hz：推定 flash frequency（BPM / 60）
    • start - end：該当区間
    一般的に、3 Hz 以上の点滅は photosensitive epilepsy の観点から注意対象とされています。
    ただし、osu!taiko の kiai effect は fullscreen strobe ではないため、 本ツールではより高い BPM を基準として確認対象を表示しています。
    ■ 注意点
    この機能は、epilepsy warning の必要性を補助的に確認するための機能です。
    表示された結果が、そのまま warning 必須・unrankable を意味するわけではありません。

    実際の視覚刺激の強さは：
    • flash の面積
    • brightness / contrast
    • flash の継続時間
    • storyboard / video の内容
    • プレイヤー環境
    などによって大きく変化します。

    特に storyboard や video の flash は、本ツールでは解析していません。
    最終的には実際のプレイ映像や storyboard 内容を確認した上で判断してください。

    スプレッド比較：Diff順

    ■ 背景
    mapset 内の複数 difficulty が適切な難易度順・難易度差で構成されているかを確認するための機能です。
    主に、低難易度から高難易度に向かって、OD / HP、ノーツ数などが自然に変化しているかを一覧で確認することを目的としています。
    ■ 検出内容
    Diff順は、difficulty 名から自動推定されます。

    自動推定できない difficulty は Unknown として表示されます。
    Unknown は必ず手動確認が必要なため、Diff順タブでは赤色で表示されます。

    Diff順は、以下の方法で手動調整できます：
    • カテゴリのプルダウンから、Kantan / Futsuu / Muzukashii 等を選択する
    • ↑ / ↓ ボタンで同カテゴリ内の順番を微調整する
    この手動調整後の順番は、OD / HP、ノーツ数、フィニッシャーの各表示にも反映されます。
    ■ 注意点
    Guest difficulty やカスタム難易度名が含まれる場合、Diff 順の自動推定が完全ではないため、Diff順やカテゴリは手動で調整する必要があります。

    Spread チェックは、自動的に unrankable を断定する機能ではありません。
    特に、Guest difficulty が含まれる場合は spread gap が許容されやすいことには注意が必要です。

    また、ノーツ数、OD / HP、フィニッシャー配置は spread の一部の要素に過ぎません。
    実際の難易度差は、リズム、密度、BPM、SV、パターン、hand usage、stream / burst の長さなどにも大きく影響されます。

    最終的には譜面内容を確認した上で判断してください。

    スプレッド比較：OD・HP

    スプレッド比較：ノーツ密度

    ■ 背景
    総ノーツ数が難易度順に増えていても、曲中の一部区間では、下位 difficulty の方が上位 difficulty よりもノーツ密度が高くなっている場合があります。

    このチェックは、そのような局所的なノーツ密度の逆転を確認するための機能です。
    ■ 検出内容
    各 difficulty について、1小節ごとの Circle ノーツ数（Slider / Spinner は含めません）を計算し、隣接する difficulty 間で比較します。

    同じ小節内で、下位 difficulty のノーツ数が上位 difficulty のノーツ数を上回っている場合、確認対象として表示します。
    ■ 注意点
    このチェックは、局所的なノーツ密度の逆転を確認するための補助機能です。
    表示された結果が、そのまま unrankable や問題を意味するわけではありません。

    以下のような理由で、一時的に下位 difficulty の方が高密度になる場合があります。
    • 上位 difficulty で休符を作り、別区間で密度を上げている場合
    • 下位 difficulty が主旋律を取り、上位 difficulty が別の音取りをしている場合
    • Guest difficulty が含まれている場合
    そのため、表示された小節については、実際の音取りや譜面構成を確認した上で判断してください。

    また、小節境界付近の 1 ms 程度のズレによって、ノーツが前後の小節に誤って分類されることを防ぐため、小節境界から ±3 ms 以内のノーツは境界側に補正して扱います。

    スプレッド比較：フィニッシャー

    ■ 検出内容
    全 difficulty の Finisher 配置を比較し、Finisher が付いているタイムポイントを一覧表示します（1ms ズレは同一時刻として扱っています）。

    表示では、以下の記号を使用します：
    • D：Don の Finisher
    • K：Kat の Finisher
    • -：ノーツがない or 小ノーツ or スライダー or スピナー
    Finisher の表は、以下の3種類に分けて表示されます：
    • フィニッシャー欠損：一部の difficulty に Finisher が存在しない
    • D/K 不一致：全 difficulty で Finisher ではあるが、D / K の種類が一致していない
    • 完全一致：全 difficulty で Finisher であり、D / K の種類も一致している
    このタブでは警告判定は行わず、比較しやすい形で情報を表示することを目的としています。

    スプレッド比較：スクロール速度

    ■ 背景
    osu!taiko Ranking Criteria では、 低難易度である Kantan / Futsuu / Muzukashii において、急激な Slider Velocity 変化や読み取りにくいスクロール変化を避けるべきとされています。

    低難易度で視認性を損なうような急激なスクロール速度変化がないかを確認します。

    また、スプレッド全体としてのスクロール速度も解析し、以下のような事項も確認します。
    • difficulty が上がるにつれて、スクロール速度が不自然に逆転していないか
    • 同じ時間帯のスクロール変化が、difficulty ごとに大きく矛盾していないか（加速 or 減速等）
    ■ 検出内容 1：スクロール速度
    単純な SV 値だけでは、BPM や SliderMultiplier の影響を考慮できないため、
    このチェックでは実際の見た目に近いスクロール速度を比較するために、有効スクロール速度（effective scroll speed, eSV ）を使用します。
    effective scroll speed = 175 × SliderMultiplier × SV × BPM / 60
    有効スクロール速度は、「ノーツが1秒間に画面上を何ピクセル移動するか」を表す値です。
    単位は px/s（pixels per second）です。

    表示内容は以下の通りです。
    • Min px/s：そのdiff内で最も遅い有効スクロール速度
    • Max px/s：そのdiff内で最も速い有効スクロール速度
    • Delta：最大速度と最小速度の差
    • Ratio：最大速度 / 最小速度
    • SV range：そのdiff内で使用されているSVの範囲

    実際にノーツが存在する区間のスクロール速度のみを対象としています。
    ノーツが存在しない区間のSV変更は、演出用SVなどによる誤検出を防ぐため無視されます。

    隣接するノーツ間でスクロール速度が急激に変化している箇所を検出します。
    ■ 警告条件 1
    下位 difficulty において、急激なスクロール速度変化が発生している箇所を検出します。
    • eSV差：eSV after - eSV before
    • eSV倍率：eSV after / eSV before
    • ノーツ間隔：変化前後のノーツ間の時間差
    Difficulty eSV差 eSV倍率 ノーツ間隔 判定
    Kantan 以下 / Kantan ≥ 80 px/s ≥ x1.15 ≤ 1500 ms Warning
    Futsuu ≥ 80 px/s ≥ x1.15 ≤ 1250 ms Warning
    Muzukashii ≥ 160 px/s ≥ x1.25 ≤ 1000 ms Warning
    Oni / Inner Oni 以上 - - - 対象外
    上記の3条件をすべて満たした場合のみ Warning として表示します。

    eSV差とeSV倍率は、それぞれ異なる種類の急変を検出するために使用します。
    例えば、eSV差が大きくてもeSV倍率が小さい場合は「高速帯での微調整」である可能性があり、
    逆にeSV倍率が大きくてもeSV差が小さい場合は「低速帯での小変化」である可能性があります。

    そのため、このチェックでは「eSV差」と「eSV倍率」の両方を満たした場合のみ Warning として扱います。

    ノーツ間隔は、「どれくらい急激に速度が変化したか」を判定するために使用します。
    例えば、同じ速度差でも、数秒かけてゆっくり変化している場合は Warning になりません。

    この機能のアイデアは、Hivie による MVTaikoChecks の feature request を参考にしています。
    参考： Add a drastic SV change warning for lower difficulties

    この feature request では、Hivie が数名の BN と相談した結果として、 lower difficulty における急激な SV 変化の目安に以下の値が挙げられています：
    • Kantan / Futsuu：SV差 > 0.1
    • Muzukashii：SV差 > 0.2
    • Oni：SV差 > 0.3
    ただし、本ツールでは Ranking Criteria の改定により、Oni 以上の difficulty はこのチェックの対象外としています。

    例えば BPM 180 、SliderMultiplier 1.4 の場合、 SV 差 0.1 は約 73.5 px/s、 SV 差 0.2 は約 147 px/s に相当します。
    そのため、本ツールではそれらに近い実用的な値として、 Kantan / Futsuu では 80 px/s、 Muzukashii では 160 px/s を使用しています。

    maxGapMs については、difficulty が上がるほど一般的にノーツ間隔が短くなるため、 Kantan = 1500 ms、 Futsuu = 1250 ms、 Muzukashii = 1000 ms というように、段階的に短くしています。
    ■ 注意点
    この判定は、本ツール上で便宜的に設定した目安です。
    Warning が表示された場合でも、それ自体が unrankable を意味するわけではありません。

    スクロール速度変化が曲の展開・音取り・休憩・演出に対応している場合は、問題ない可能性があります。
    最終的には、実際の譜面内容とプレイ時の読みやすさを確認して判断してください。

    また、Oni 以上では大きなSV変化が演出や難易度表現として使われることが多いため、このチェックでは Warning 対象外としています。
    ■ 警告条件 2
    有効スクロール速度の不自然な Progression を検出します 有効スクロール速度が difficulty 順に一方向へ増加・減少、または同じ値のまま進行している場合は問題なしとします。

    例えば、以下のような progression は問題なしとして扱います。
    • 520 → 610 → 720 → 800：difficulty が上がるにつれて速度が上がっている
    • 800 → 720 → 610 → 520：difficulty が上がるにつれて速度が下がっている
    • 640 → 640 → 640 → 640：全 difficulty で同じ速度
    一方で、以下のように途中で進行方向が反転する場合は確認対象として表示します。
    • 610 → 750 → 680 → 820：増加した後に一度減少している
    • 800 → 700 → 760 → 650：減少した後に一度増加している
    difficulty ごとにノーツ配置や timing が完全一致しない場合があるため、±10 ms 以内に発生したスクロール速度変化を、同じタイミングの変化として扱います。

    この機能のアイデアは、Nostril による MVTaikoChecks の feature request を参考にしています。
    参考： MVTaikoChecks issue #32
    ■ 検出内容 2：スクロール変化の整合性
    各 difficulty において、隣接するノーツ間の有効スクロール速度（eSV）の変化量を計算し、difficulty 間で比較します。

    この機能は：
    • 下位 difficulty の方が上位 difficulty より強い局所スクロール変化を持っていないか
    • 同じタイミング付近で、difficulty ごとにスクロール変化の方向が不自然に異なっていないか
    を確認するための補助機能です。

    表示内容は以下の通りです。
    • Δ px/s：隣接ノーツ間で発生した eSV の変化量（eSV after - eSV before）
    • Speed Multiplier：eSV の変化前後の値と倍率。
      例：420 → 735 (x1.75)
    • Note Interval：eSV 変化が発生した2つのノーツ間の時間差
    例えば、下位 difficulty では 420 → 735 のような大きな加速がある一方で、上位 difficulty では同じタイミング付近の変化が小さい場合、確認対象として表示されます。

    また、同じタイミング付近で、一方の difficulty は加速、もう一方の difficulty は減速している場合も確認対象になります。
    ■ 警告条件 3
    下位 difficulty の方が、上位 difficulty より強い局所スクロール変化を持っているパターンを検出します。

    このチェックでは、隣接ノーツ間で発生した有効スクロール速度変化（Δ px/s）を difficulty 間で比較します。
    下位 difficulty の eSV 変化量が、上位 difficulty より明らかに大きい場合に確認対象として表示します。
    項目 条件 内容
    同一タイミング判定 ±10 ms 近いタイミングで発生したスクロール変化を、同じタイミングの変化として扱います。
    比較対象 隣接する difficulty Diff順に並べたうえで、隣同士の difficulty を比較します。
    小さい変化の除外 |Δ px/s| < 80 非常に小さい eSV 変化は、微小な SV 調整によるノイズを避けるため比較対象から除外します。
    下位 difficulty の変化が大きすぎる場合 lower |Δ| ≥ higher |Δ| × 1.5
    かつ lower |Δ| - higher |Δ| ≥ 120 px/s     		下位 difficulty の局所スクロール変化が、上位 difficulty より明らかに強い場合に表示します。
    ■ 警告条件 4
    同じタイミング付近において、difficulty ごとにスクロール速度変化の方向が一致していないパターンを検出します。

    このチェックでは、隣接ノーツ間で発生した eSV 変化（Δ px/s）を difficulty 間で比較します。
    同じタイミング付近で、一方の difficulty は加速、もう一方の difficulty は減速している場合に確認対象として表示します。
    項目 条件 内容
    同一タイミング判定 ±10 ms 近いタイミングで発生したスクロール変化を、同じタイミングの変化として扱います。
    比較対象 隣接する difficulty Diff順に並べたうえで、隣同士の difficulty を比較します。
    変化方向の不一致 両方の |Δ px/s| ≥ 120
    かつ Δ の符号が逆     		同じタイミング付近で、一方は加速、もう一方は減速している場合に表示します。
    「変化方向の不一致」では、体感差の小さい微小変化を除外するため、両方の eSV 変化量が十分大きい場合のみ比較対象としています。
    ■ 注意点
    この判定は、本ツール上で便宜的に設定した目安です。

    スクロール変化の整合性チェックは、同じ時間帯の変化イベントを自動的に対応付けて比較します。
    そのため、diffごとにSV変更の位置が大きく異なる場合、意図した比較にならない可能性があります。

    また、下位 difficulty の方が強い局所スクロール変化を持っている場合でも、演出・音取り・休憩配置などの理由で妥当な場合があります。
    表示された結果は Warning ではなく、確認対象として扱ってください。

    BN評価

    ■ 背景
    一部の BN / NAT 専用の機能です。

    BN evaluation において、mod適用前の .osz と mod適用後の .osz を比較し、どこが変更されたかを確認するための機能です。
    diff名が変更されている場合もあるため、読み込み後に Before / After の diff を手動で対応付けてから比較します。
    ■ 検出内容
    以下の5種類の情報を比較します。
    • ノーツ情報：ノーツの追加、削除、種類変更、スライダー / スピナーの追加・削除、スライダー / スピナーのテール移動を表示します。
    • タイムライン： 変更がある小節のみをタイムライン形式で表示します。
      小節内の最小 snap に応じて自動的にグリッド幅を変更します。
    • Timing情報：BPM / SV の追加・削除・変更、Volume の相違区間、Kiai の相違区間を表示します。
    • メタデータ情報： メタデータの変更点を表示します。
    • OD / HP：OD /HP の変更を表形式で表示します。
    ノーツについては、 d / k / D / K / Slider / S / Spinner / S の差分を表示します。

    なお、1 msズレのノーツ移動は表示しない仕様です。
    ただし、1 msズレていても d → k のように内容が変更されている場合は、変更として表示されます。
    ■ 注意点
    Before / After で全体のオフセットが変わる場合は、赤線を元にオフセット補正 を使用できます。
    この機能では Before と After の最初の赤線の位置を比較し、その差分を offset として計算します。

    ただし、NC rest の追加などにより赤線の位置が 1/1 拍単位などでずれる場合は対応できません。
    その場合は、最初の赤線の位置を揃えてから再度比較してください。
