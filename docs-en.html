Modding Helper Documentation

Author: Vanity8

This tool's snap detection is based on beatLength and does not consider meter (time signature).
Therefore, 1/1 snap refers to "each beat position," not the beginning of a measure.

Clap / Whistle

■ Background
There are two types of Kat sounds (Clap / Whistle).
Some BN users prefer not to use Whistle, so unifying to Clap may be recommended.
■ Detection
Analyzes hitsounds of Circle objects (type=1/5) and displays counts of Clap / Whistle / Both.
When enabled, timestamps for each category are also displayed.
■ Notes
Using Whistle or Whistle + Clap is not necessarily unrankable.

1ms Offset

■ Background
When 60000 / BPM does not result in an exact integer, floating-point errors may occur during copy & paste in the osu! editor, causing a 1 ms offset.
Such offsets can lead to SV not applying correctly or inaccuracies in SR calculation.
Mapset Verifier and AiMod do not detect ±1 ms offsets (as they are not unrankable).
■ Detection
Detects HitObjects that are offset by ±1–3 ms from their expected snap positions.
Snap positions are calculated based on beatLength, and decimals are truncated (matching osu! editor behavior).
The following snap values are checked: 1/1, 1/2, 1/3, 1/4, 1/5, 1/6, 1/7, 1/8, 1/9, 1/12, 1/16.
Other snap values are not included.
■ Notes
±1 ms offsets are not unrankable, so not all cases need to be fixed.

Kiai Compare

■ Background
When total kiai duration differs between difficulties, it can be difficult to identify where the mismatch occurs.
Mapset Verifier can compare total duration but cannot show the exact mismatching sections.
■ Detection
Displays sections where kiai states do not match.
Also warns when kiai end points are not explicitly defined.
■ Notes
Different kiai patterns across difficulties are not unrankable, especially in Guest difficulties.
However, consistency is recommended when the same mapper created multiple difficulties.

Kiai Snap

■ Background
AiMod can detect unsnapped timing points, but cannot detect misalignment at specific snap levels (e.g. 1/4, 1/8).
■ Detection
Detects kiai ON/OFF points that are not aligned to 1/1 snap.
Also detects unsnapped kiai lines (e.g. 1/1 snap + 1 ms).

Double SV

■ Background
AiMod can detect overlapping SV lines at the same timestamp, but cannot detect SV lines that are slightly offset (e.g. 1 ms).
■ Detection
Detects SV lines that are too close to each other within a specified interval (1–5 ms).

SV Volume

■ Background
If a volume-changing SV line is placed at the same timestamp as a HitObject, the new volume may not be applied if the player hits the note 1 ms early.
Therefore, it is recommended to place such SV lines at least 5 ms before the HitObject (commonly 5 ms or 1/16 snap earlier).
This check is not available in AiMod or Mapset Verifier.
■ Detection
Warns when volume-changing SV lines are placed too close to HitObjects.

Volume Compare

■ Detection
Compares SV volume across difficulties and detects sections where they do not match.
When "Show only differences of 5% or more" is enabled, only differences of 5% or greater are displayed.

Red & Green Match

■ Background
Red lines (BPM) and green lines (SV) at the same timestamp should generally share the same settings.
This check is not available in AiMod or Mapset Verifier.
■ Detection
Detects mismatches in volume, kiai, and sampleset.
■ Notes
The first red line cannot have kiai enabled.
Therefore, if the first SV line has kiai enabled, it will be flagged.
Also, volume differences at the first timing point may be intentional.
Before that point, the red line volume is used; after that point, the green line volume is applied.

SampleSet

■ Background
Each BPM line, SV line, and HitObject can have a sampleset assigned.
This defines the type (tone) of hitsound used:
  • 1: Normal
  • 2: Soft
  • 3: Drum
In osu!taiko, maps are generally expected to use only Normal (1).
Other values may indicate unintended hitsounds.
This check is not available in AiMod or Mapset Verifier.
■ Detection
Detects non-Normal samplesets and custom hitsounds.

For TimingPoints:
  • sampleset ≠ 1 (normal) → 2: Soft / 3: Drum
  • sampleIndex ≠ 0 → custom hitsound
For HitObjects:
  • normalSet > 1 → 2: Soft / 3: Drum
  • additionSet > 1 → 2: Soft / 3: Drum
  • sampleIndex ≠ 0 → custom hitsound
  • filename is specified → custom audio
For slider edges:
  • edgeSets.normalSet > 1 → 2: Soft / 3: Drum
  • edgeSets.additionSet > 1 → 2: Soft / 3: Drum

Slider Multiplier / TickRate

■ Background
In osu!taiko, SliderMultiplier should generally be set to 1.40.
Reference: osu!taiko Ranking Criteria
SliderTickRate is normally set to 1, while 3 is generally used for swing-rhythm songs.
■ Detection
This check warns when SliderMultiplier is not 1.40.
It also calculates the ratio of HitObjects placed on 1/3 snap. If the 1/3 snap ratio is 30% or higher, the map is treated as swing-rhythm for this check.
When the map is treated as swing-rhythm, SliderTickRate = 3 is recommended; otherwise, SliderTickRate = 1 is recommended. A warning is shown when the actual value does not match the recommended value.
■ Notes
For high-BPM low difficulties, SliderMultiplier values other than 1.4 may be used as exceptions (see Taiko Checklist V1.2).
Also, the 30% threshold for the 1/3 snap ratio is an arbitrary value chosen for this tool.
Therefore, some swing-rhythm maps may fall below this threshold, and some non-swing maps may exceed it.
Use this result as a guide, and make the final judgement based on the actual rhythm and mapping context.

Tag

■ Background
Tags must be separated by exactly one half-width space.
AiMod and Mapset Verifier cannot detect cases where multiple half-width spaces or full-width spaces are used.
■ Detection
Warns when two or more half-width spaces or any full-width spaces are found between tags.

BN Review

■ Background
This is a feature mainly intended for some BN / NAT use cases.
During BN evaluation, it compares the .osz before mod application and the .osz after mod application, making it easier to check what was changed.
Since difficulty names may be changed during the modding process, Before / After difficulties are matched manually after loading both files.
■ Detection
This feature compares the following three categories:
  • Notes: shows added notes, deleted notes, note type changes, added/deleted sliders and spinners, and slider/spinner tail movement.
  • Timing: shows added/deleted/changed BPM or SV lines, volume mismatch sections, and kiai mismatch sections.
  • Metadata: shows changes in Artist, Romanised Artist, Title, Romanised Title, Source, and Tags.
Tags are compared as space-separated entries, and removed / added tags are displayed separately.
For notes, differences are shown using d / k / D / K, Slider, and Spinner labels.
Note that 1 ms note movement is intentionally hidden. However, if the note content changes at the same time, such as d → k, it will still be shown as a change.
■ Notes
If the global offset differs between Before and After, you can use “Apply offset from first red line”.
This calculates the offset difference by comparing the first red line position in Before and After.
However, it cannot handle cases where the red line position is shifted by a musical amount such as 1/1 beat due to adding an NC rest or similar edits.
In that case, align the first red line positions first, then try comparing again.
