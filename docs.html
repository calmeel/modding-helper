    このツールについて

    ■ 概要
    このツールは、ranked譜面を作成する mapper や、modding を行う Beatmap Nominator 向けの osu!taiko modding 支援ツールです。
    また、一部には BN evaluation を行う NAT 向けの機能も実装しています。
    ■ 背景
    Ranked譜面を作成する際には、一般的に AiMod、 Mapset Verifier、 Mapset Verifier MVTaikoChecks などのチェックツールを使用します。しかし、これらのツールで検出できる内容は完全ではありません。

    例えば Taiko Checklist V1.2 by iRedi を見ると、これらのツールでは検出できないチェック項目も存在することが分かります。
    本ツールは、主に Mapset Verifier ではチェックできない項目を自動で確認するためのツールです。
    基本的には、Mapset Verifier と本ツールの両方を併用することを想定しているため、Mapset Verifier で検出可能な項目は本ツールでは基本的に検出しません。

    また、本ツールには一部の AiMod に近い機能も実装しています。
    特に、複数diffをまとめて確認できるため、「全diffを一括で確認できる AiMod」のような使い方ができます。
    ただし、AiMod のすべての機能を実装しているわけではないため、最終的には osu!editor 上で AiMod による確認も行うことを推奨します。

    最後に、本ツールと Mapset Verifier を併用しても、すべての Ranking Criteria 違反を検出できるわけではありません。
    どうしても人間の目で確認する必要がある項目もあるため、自動チェックの結果だけに頼りすぎないよう注意してください。
    ■ 参考
    • Ranking Criteria
    • osu!taiko Ranking Criteria
    ■ 便利な設定
    • 使用しないチェック項目は、画面右上の歯車ボタンから非表示にできます。
    • チェックボックスやタブの表示設定はブラウザに保存されます。
      そのため、次回アクセス時も前回と同じ設定で使用できます。
    ■ 注意事項
    • このツールの snap 判定は beatLength を基準にしており、meter（拍子）は考慮していません。
      そのため、1/1 snap は小節頭ではなく「1拍ごとの位置」を意味します。
    • ハイブリッド mapset の場合は、taiko モードの diff のみを表示します。
    • 1 ms単位での正確な snap 推定は、beatLength を用いて計算し、最終的に小数点以下を切り捨てることで行っています。これにより、osu!editor とほぼ同じ位置の snap を推定できます。
    • ブラウザでページを開いたままにしている場合、ツールの更新が自動では反映されないことがあります。
      定期的にページをリロードするか、更新が反映されない場合は Ctrl + F5 で強制リロードしてください。

    Clap / Whistle

    ■ 背景
    Kat には 2種類（Clap / Whistle）があります。
    一部のBNは Whistle の音を好まないため、Clap に統一するのが望ましい場合があります。
    ■ 検出内容
    Circle（type=1/5）の hitsound を解析し、Clap / Whistle / 両方 の個数を表示します。
    チェックボックスを有効にすると、それぞれのタイムポイントも表示されます。
    ■ 注意点
    Whistle、Whistle + Clap の使用は unrankable ではありません。

    1msズレ

    ■ 背景
    60000 / BPM が割り切れない場合、osu!editorでのコピーペーストにより浮動小数誤差が発生し、1 ms のズレが生じることがあります。
    1 msズレは SV線の未適用や SR計算の誤差につながるため、なるべく避けたほうが良いです。
    Mapset Verifier や AiMod では ±1 ms のズレは表示されません（unrankable ではないため）。
    ■ 検出内容
    HitObject（ノーツ・スライダー・スピナー）が snap 位置から ±1〜3 ms ずれている箇所を検出します。
    終点についてはスピナー終点のズレのみ検出します（スライダー終点のズレ検出は技術的制約により未実装です）
    このツールでは、.osu ファイルの beatLength をもとに snap 位置を計算し、小数点以下を切り捨てて比較しています（osu!editorと一致するはずです）。
    チェック対象の snap は、1/1, 1/2, 1/3, 1/4, 1/6, 1/8, 1/12, 1/16 です。
    それ以外の snap は検出対象外です。

    なお、1/5, 1/7, 1/9 snap は、オプション（チェックボックス）を有効にした場合のみ検出されます。
    ■ 注意点
    ±1 ms は unrankable ではないため、必ずしも全て修正する必要はありません。

    Kiai比較

    ■ 背景
    diffごとに kiai 総時間が異なる場合、どの区間で差があるのか分かりにくいことが多いです。
    Mapset Verifier では総時間の比較はできますが、具体的にどの区間が異なるかは確認できません。
    ■ 検出内容
    kiai 状態が一致していない区間を表示します。
    また、kiai time の終了地点が明示されていない場合も警告を行います（AiModと同一の機能）。
    ■ 注意点
    diffごとに kiai を変えること自体は unrankable ではありません。特に Guest difficulty ではよく見られます。
    ただし、同一作者の場合は特別な理由がない限り揃えることが推奨されることが多いです。

    Kiaiスナップ

    ■ 背景
    AiMod は unsnapped な timing point を検出できますが、1/4 や 1/8 snap など、snap単位でズレている場合は検出できません。
    ■ 検出内容
    kiai の ON/OFF が 1/1 snap 以外に乗っている場合を検出します。
    また、unsnapped な kiai line（1/1 snap + 1 ms など）も検出します（AiModと同一の機能）。

    二重SV

    ■ 背景
    二重 SV はどちらかが意図しない or 消し忘れであるケースが多いです。 AiMod は同時刻の重複 SV は検出できますが、1 ms ズレの二重 SV は検出できません。
    ■ 検出内容
    指定した間隔（1〜5 ms）以内にある、近すぎる SV line を検出します。

    SV ボリューム

    ■ 背景
    HitObject と同時刻に volume変更を置いた場合、プレイヤーが 1 ms 早く叩くと新しい volume が適用されない場合があります。
    そのため、volume変化を伴うSV線は、HitObject に対して 5 ms 以上前倒しして設置することが推奨されています（一般的には 5 ms, 1/16, 1/8 snap 程度ズラす事が多いです）。
    ■ 検出内容
    ボリューム変更を伴う SV line が、HitObject と近すぎる場合に警告します（この検出は AiMod や Mapset Verifier では行えません）。 検出閾値は、「5ms（5ms 丁度は含まない）」「1/16 snap（1/16 snap 丁度は含まない）」から選択できます。 「15% 以上の差のみ表示」のチェックボックスを有効にすると、volume 差が 15% 以上の際のみ表示します。

    ボリューム比較

    ■ 検出内容
    diff間の SV ボリュームを比較し、不一致がある区間を検出します（この検出は AiMod や Mapset Verifier では行えません）。
    「5% 以上の差のみ表示」を有効にすると、volume 差が 5% 以上の区間のみを表示します。

    赤線 & 緑線の一致

    ■ 背景
    同時刻の赤線（BPM line）と緑線（SV line）は、基本的に同じ設定である必要があります。
    ■ 検出内容
    ボリューム / kiai のオンオフ / sampleset の不一致を検出します（この検出は AiMod や Mapset Verifier では行えません）。
    ■ 注意点
    最初の赤線は kiai ON にすることができません。そのため、最初の SV line が kiai ON の場合は、このチェックに引っかかります。
    最初の赤線と緑線では、意図的に volume を変更している場合があります。そのタイムポイントより前は赤線の volume が適用され、それ以降は緑線の volume が適用されるためです。

    SampleSet

    ■ 背景
    BPM線（赤線）やSV線（緑線）、および各 HitObject には、sampleset と呼ばれる効果音の種類を割り当てることができます。
    sampleset はヒット音の「音色」を決める設定で、主に以下の種類があります：
    • 1: Normal
    • 2: Soft
    • 3: Drum
    osu!taiko モードでは、大抵の場合 1: Normal のみを使用するため、それ以外の sampleset が含まれている場合は、意図しない効果音が混入している可能性があります。
    この検出は AiMod や Mapset Verifier では行えません。
    ■ 検出内容
    Normal 以外の sampleset やカスタムヒットサウンドを検出します（この検出は AiMod や Mapset Verifier では行えません）。

    赤線・緑線では、以下を検出します：
    • sampleset ≠ 1 (normal) → 2: Soft / 3: Drum
    • sampleIndex ≠ 0 → カスタムヒットサウンド
    ノーツ、スライダー、スピナーでは、以下を検出します：
    • normalSet > 1 → 2: Soft / 3: Drum
    • additionSet > 1 → 2: Soft / 3: Drum
    • sampleIndex ≠ 0 → カスタムヒットサウンド
    • filename が指定されている → カスタム音源
    スライダーの各エッジに設定されたヒットサウンドも検出します：
    • edgeSets.normalSet > 1 → 2: Soft / 3: Drum
    • edgeSets.additionSet > 1 → 2: Soft / 3: Drum

    スライダー感度・目盛り

    ■ 背景
    osu!taiko では、スライダー感度（SliderMultiplier）は基本的に 1.40 に設定する必要があります。
    参考： osu!taiko Ranking Criteria
    また、スライダー目盛り（SliderTickRate）は通常 1 を用いますが、swingリズムの曲では 3 を用いるのが基本です。
    ■ 検出内容
    SliderMultiplier が 1.40 ではない場合に警告します（この検出は AiMod や Mapset Verifier では行えません）。
    全ての HitObject の位置から 1/3 snap の割合を計算し、1/3 snap の割合が 30% 以上の場合は swingリズムとして扱います。
    swingリズムと判定された場合は SliderTickRate = 3、それ以外の場合は SliderTickRate = 1 を推奨値として表示し、実際の値と異なる場合に警告します（この検出は AiMod や Mapset Verifier では行えません）。
    ■ 注意点
    High BPM の low diff では、例外的に SliderMultiplier = 1.4 以外を用いることがあります（参照：Taiko Checklist V1.2 by iRedi）。
    また、「1/3 snap の割合が 30% 以上」という基準は、このツール上で便宜的に設定した値です。
    そのため、この値を下回っている swingリズムの map や、この値を上回っている通常リズムの map が存在する可能性があります。
    最終的には、曲のリズムや譜面内容を確認した上で判断してください。

    プチフリ

    ■ 背景
    osu! では、音源の読み込み時に一瞬のフリーズ（プチフリーズ）が発生する場合があります。
    音源開始直後に最初のノーツが配置されていると、ノーツが画面に表示されている最中、または判定枠に近い位置でこのフリーズが発生し、正しいリズムで叩くことが難しくなる可能性があります。

    そのため、最初のノーツが音源開始直後に置かれている場合は、音源ファイルの先頭に 500〜1000 ms 程度の無音を挿入することが一般的に推奨されています。
    ■ 検出内容
    最初のノーツが、音源開始時点で画面内のどの位置に表示されているかを推定します。
    このチェックでは、最初の Circle オブジェクトのみを対象とし、Slider や Spinner は検出対象外です。

    判定には、最初のノーツ時点で有効な BPM line（赤線）と SV line（緑線）を使用します。
    BPM と SV から osu!taiko のスクロール速度を推定し、16:9 環境を基準として、ノーツが画面に入ってから判定枠に到達するまでの時間を計算します。

    結果には、以下の情報を表示します：
    • Visible：ノーツが画面に入ってから判定枠に到達するまでの推定時間
    • Remain：音源開始時点で、判定枠までどの程度残っているかの推定割合（音源開始時点でのノーツ位置）
    ■ 判定基準
    • Warning：音源開始時点で、最初のノーツが表示時間の半分以内の位置にある場合（Remain ≤ 50%）
    • Error：音源開始時点で、最初のノーツが判定枠にかなり近い位置にある場合（Remain ≤ 25%）
    ■ 注意点
    この判定は、16:9 環境を基準にした推定です。
    実際の見え方は、画面比率、スキン、HR の有無、osu!stable / osu!lazer の違い、PC環境などによって変わる可能性があります。

    また、プチフリの発生タイミングや強さは環境依存です。
    このチェックは「必ず問題が起こる」ことを示すものではなく、音源開始直後のノーツ配置が危険かどうかを確認するための目安として使用してください。

    Tag

    ■ 背景
    Tag と Tag の間は、"半角スペース1つ"で区切る必要があります。
    AiMod や Mapset Verifier では、"2つ以上の半角スペース"や"全角スペース"が含まれている場合検知できません。
    ■ 検出内容
    まず、全diffで Tags が一致しているかを確認します（Mapset Verifier と同一の機能）。
    その後、各diffの Tags に対して、以下のスペース問題を検出します：
    • Tag と Tag の間に、2つ以上の半角スペースが含まれている場合
    • Tag と Tag の間に、全角スペースが含まれている場合
    • スペルミスの可能性がある tag
    • 関連tag / abbreviation の追加候補
    スペルチェックでは、主にジャンル名や mapping 関連の tag を対象に、1文字違い・1文字抜け・1文字多い可能性がある tag を検出します。
    例：hardcroe → hardcore, vocaliod → vocaloid

    関連tag / abbreviation 候補では、特定の tag が含まれている場合に、併記されることが多い tag を追加候補として表示します。
    例：jpop → j-pop, pop / tiebreaker → tb

    この検出は AiMod や Mapset Verifier では行えません。
    ■ 注意点
    スペルチェックや関連tag候補は、必ず修正・追加が必要なエラーではありません。
    特にジャンル名や abbreviation は譜面内容や楽曲情報によって適切な tag が変わるため、最終的には mapper / modder / BN が内容を確認して判断してください。
    また、対応している tag の種類は限定的であり、全てのスペルミスや関連tagを検出できるわけではありません。

    BN評価

    ■ 背景
    一部の BN / NAT 専用の機能です。
    BN evaluation において、mod適用前の .osz と mod適用後の .osz を比較し、どこが変更されたかを確認するための機能です。
    diff名が変更されている場合もあるため、読み込み後に Before / After の diff を手動で対応付けてから比較します。
    ■ 検出内容
    以下の3種類の情報を比較します。
    • ノーツ情報：ノーツの追加、削除、種類変更、スライダー / スピナーの追加・削除、スライダー / スピナーのテール移動を表示します。
    • Timing情報：BPM / SV の追加・削除・変更、Volume の相違区間、Kiai の相違区間を表示します。
    • メタデータ情報：Artist、Romanised Artist、Title、Romanised Title、Source、Tags の変更を表示します。
    Tags は半角スペース区切りで比較し、削除された tag と追加された tag を分けて表示します。
    ノーツについては、d / k / D / K、Slider、Spinner の差分を表示します。
    なお、1 msズレのノーツ移動は表示しない仕様です。ただし、1 msズレていても d → k のように内容が変更されている場合は、変更として表示されます。
    ■ 注意点
    Before / After で全体のオフセットが変わる場合は、「赤線を元にオフセット補正」を使用できます。
    この機能では、Before と After の最初の赤線の位置を比較し、その差分を offset として計算します。
    ただし、NC rest の追加などにより赤線の位置が 1/1 拍単位などでずれる場合は対応できません。
    その場合は、最初の赤線の位置を揃えてから再度比較してください。
